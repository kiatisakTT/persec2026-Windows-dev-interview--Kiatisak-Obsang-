"ในโปรเจกต์นี้ ผมเน้นการเขียนโค้ดที่ อ่านง่าย Maintainต่อได้  และ ตรวจสอบได้จริงครับ 
ผมเลยออกแบบเป็น Architecture แบบ Service-Oriented โดยแยกแต่ละโจทย์ออกมาเป็น Service แยกจากกัน 
เพื่อให้ทางผู้ตรวจสอบเข้ามาอ่านโค้ดแล้วเข้าใจทันทีว่าส่วนไหนทำอะไร"

โดยแบ่ง Service ออกเป็น 5 file ดังนี้ 
1. BracketService  => โจทย์ข้อ1
2.SortingService  => โจทย์ข้อ2 และ ข้อ 5
3.AutocompleteService => โจทย์ข้อ 3
4.RomanNumeralService => โจทย์ข้อ 4
5.TribonacciService  =>  โจทย์ข้อ 6

(Clean Architecture) และไม่อยากให้ตัวทดสอบไปกวนการทำงานของตัวโปรแกรมหลักครับ
